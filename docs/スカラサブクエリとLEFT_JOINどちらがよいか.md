# スカラサブクエリとLEFT JOIN、どっちがよいんだろうか？

スカラサブクエリはスカラサブクエリをどれだけカラムで使うか次第。例えば、10カラム分スカラサブクエリがあったら、10回SELECTが動く。
100万行SELECTするとしたら、100万行×10回、すまり1000万回サブクエリが実行される。
LEFT JOINは？おそらくだが、イイ感じにJOINしてくれるはず。
つまり、オンライン画面などでは、せいぜい1画面に表示する行数は1000行程度と考えればスカラサブクエリがよい。
バッチ処理だと1万行とか10万行のレコードを相手にするのでLEFT JOINの方がよい。

## テストデータ作成
### コードマスタ

```
DROP TABLE T_CODE_MASTER;

CREATE TABLE T_CODE_MASTER (
  ITEM_NAME VARCHAR2(20),
  CODE      VARCHAR2(20),
  VALUE     VARCHAR2(20),
  CONSTRAINT "T_CODE_MASTER_PK" PRIMARY KEY ( "ITEM_NAME",
                                              "CODE" )
)
```

```
INSERT INTO T_CODE_MASTER (
  ITEM_NAME,
  CODE,
  VALUE
)
  SELECT
    'ITEM_'
    || TO_CHAR(A.NUM, 'FM00') AS ITEM_NAME,
    B.CODE,
    B.VALUE
  FROM
    -- 100*10の1000件がINSERTされる
         (
      SELECT
        -- 0から99の2桁の数字を生成(つまり100件)
        ( D2.DIGIT_NUM * 10 ) + D1.DIGIT_NUM AS NUM
      FROM
        T_DIGITS D1,
        T_DIGITS D2
    ) A
    CROSS JOIN (
      SELECT
        -- 0から9までの数字をベースにコードを作っている(つまり10件)
        DIGIT_NUM  AS CODE,
        -- 乱数で10桁の文字列を生成
        SUBSTR(DBMS_CRYPTO.HASH(UTL_I18N.STRING_TO_RAW(DBMS_CRYPTO.RANDOMNUMBER, 'AL32UTF8'),
                                2),
               1,
               10) AS VALUE
      FROM
        T_DIGITS
    ) B;
COMMIT;
```
